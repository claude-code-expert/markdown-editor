# Contract: 라우팅 (App Router)

## 라우트

| 경로 | 파일 | 역할 |
|------|------|------|
| `/` | `app/page.tsx` | 대시보드(파일 리스트) |
| `/editor/[docId]` | `app/editor/[docId]/page.tsx` | 에디터(docId 문서 로드) |

## 계약 규칙

| # | 규칙 | 검증 |
|---|------|------|
| R1 | `/` 진입 → 대시보드 렌더(에디터 아님) | 라우팅/스모크 |
| R2 | `/editor/[docId]` 진입 → 해당 문서 로드(FR-003·FR-004) | URL→선택 |
| R3 | 새로고침 시 같은 docId 유지(SC-003) | 새로고침 |
| R4 | 없는 docId → `/` 리다이렉트 + 안내(FR-005·SC-006) | 안전 처리 |
| R5 | `loadAll` 멱등(loaded 가드) — 두 페이지 안전 호출(R2) | 재마이그레이션 0 |
| R6 | 사이드바 문서 클릭 → `/editor/[id]` 네비게이션(URL 일치) | 네비게이션 |
| R7 | 미저장 변경 시 이동 경고(FR-010) | dirty 가드 |

## 책임 경계

- 페이지 컴포넌트가 `ensureLoaded` + `docId` 선택을 구동. EditorScreen은 렌더·편집만(선택은 URL 주도).
- 첫 문서 자동선택(M3)은 제거 — URL이 활성 문서의 단일 출처.
