# Feature Specification: 문서 흐름 — 대시보드·라우팅·폴더 저장 (M6)

**Feature Branch**: `feature/init`

**Created**: 2026-06-27

**Status**: Draft

**Input**: M6 문서 흐름 — 대시보드 파일 리스트 + `/editor/[docId]` 라우팅 + 저장 시 폴더 선택 (requirement.md §12 M6, FR-3·FR-7)

## Clarifications

### Session 2026-06-27

- Q: 앱 진입점은? → A: requirement.md §2 채택 — **`/`(대시보드, 파일 리스트)**, **`/editor/[docId]`(에디터)**.
  현재 `/`(에디터 단독)에서 대시보드 우선 진입으로 전환.
- Q: 저장 시 폴더 선택과 M3 사이드바 생성의 관계? → A: 상단 영역의 **폴더 경로 선택**으로 현재 문서의
  저장 대상 폴더를 지정한다. 사이드바의 폴더별 문서 생성(M3)과 공존하며, 폴더 미선택 시 저장을 막는다.
- Q: 중복 파일명은? → A: 동일 폴더 내 중복 제목은 경고하거나 자동 접미사(`-1`)로 구분(권장).

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 대시보드에서 문서 찾아 열기 (Priority: P1)

사용자가 앱에 진입하면 저장된 모든 문서가 리스트로 보이는 대시보드를 본다. 문서를 클릭하면 에디터로
이동해 그 문서를 편집한다.

**Why this priority**: 문서가 여러 개일 때 "무엇이 있는지 보고 고르는" 관문이 없으면 작업 시작이 막힌다.
대시보드는 다중 문서의 진입점(requirement.md FR-7.3·7.4).

**Independent Test**: 문서 2개 이상 있을 때 대시보드 진입 → 리스트에 모두 표시 → 하나 클릭 → 에디터에
그 문서가 로드되는지로 검증.

**Acceptance Scenarios**:

1. **Given** 저장된 문서 여러 개, **When** 대시보드 진입, **Then** 모든 문서가 리스트(제목·소속 폴더·수정
   시각 등)로 표시된다.
2. **Given** 대시보드 리스트, **When** 문서 클릭, **Then** 에디터 화면으로 이동해 해당 문서가 로드된다.
3. **Given** 문서가 하나도 없음, **When** 대시보드 진입, **Then** 빈 상태 안내와 새로 시작할 수단이 보인다.

---

### User Story 2 - 문서별 주소(URL) (Priority: P1)

각 문서가 고유 주소(`/editor/[docId]`)를 가져, 그 주소로 바로 들어가거나 새로고침해도 같은 문서가
열린다. 특정 문서로의 직접 이동·공유(로컬)가 가능하다.

**Why this priority**: 주소가 문서를 가리키지 않으면 새로고침 시 어떤 문서였는지 잃는다. URL은 다중 문서
네비게이션의 기반(FR-7.4).

**Independent Test**: 문서 A의 주소로 직접 진입·새로고침했을 때 항상 A가 열리고, 없는 주소는 안전하게
처리되는지로 검증.

**Acceptance Scenarios**:

1. **Given** 문서 A의 주소, **When** 그 주소로 진입, **Then** 에디터에 A가 로드된다.
2. **Given** 에디터에서 A 편집 중, **When** 새로고침, **Then** 여전히 A가 열린다.
3. **Given** 존재하지 않는 문서 주소, **When** 진입, **Then** 안내와 함께 대시보드로 돌아갈 수단이 제공된다.

---

### User Story 3 - 저장 시 폴더 선택 (Priority: P1)

사용자가 문서를 저장할 때 상단에서 대상 폴더를 선택한다. 폴더를 고르지 않으면 저장이 막히고 안내가
표시된다.

**Why this priority**: 폴더 구조에 문서가 제대로 들어가야 정리가 성립한다. 폴더 미선택 저장은 미아 문서를
만든다(FR-3.1~3.3).

**Independent Test**: 폴더 미선택 상태로 저장 시도 → 차단·안내, 폴더 선택 후 저장 → 해당 폴더에 문서가
들어가는지로 검증.

**Acceptance Scenarios**:

1. **Given** 편집 중인 문서, **When** 상단에서 폴더 경로 선택, **Then** 그 폴더가 저장 대상으로 지정된다.
2. **Given** 폴더 미선택, **When** 저장 시도, **Then** 저장이 막히고 폴더 선택 안내가 표시된다.
3. **Given** 폴더 선택 후, **When** 저장, **Then** 문서가 해당 폴더 하위에 저장되고 성공 피드백이 표시된다.

---

### User Story 4 - 중복 파일명 처리 (Priority: P3)

같은 폴더에 같은 제목으로 저장하려 하면 시스템이 경고하거나 자동으로 구분(`-1`)한다.

**Why this priority**: 편의·정합성 향상이나, 핵심 저장·열기 흐름을 차단하지 않으므로 P3.

**Acceptance Scenarios**:

1. **Given** 폴더에 "메모"가 있음, **When** 같은 폴더에 "메모" 저장, **Then** 경고하거나 "메모-1"로 구분 저장된다.

---

### Edge Cases

- 폴더가 하나도 없는데 저장하려 하면? → 폴더 생성을 유도하거나 기본 폴더로 안내한다.
- 대시보드에서 연 문서를 다른 곳에서 삭제했다면? → 없는 문서 주소 진입과 동일하게 안전 처리된다.
- 미저장 변경이 있는 상태로 대시보드/다른 문서로 이동하면? → 저장되지 않은 변경에 대해 경고한다.
- 라우팅 도입이 기존 에디터·사이드바·저장·다중문서 동작을 깨지 않아야 한다.

## Requirements *(mandatory)*

### Functional Requirements

**대시보드·라우팅**

- **FR-001**: 시스템은 앱 진입 시 저장된 문서를 리스트로 보여주는 대시보드 화면을 제공해야 한다.
- **FR-002**: 대시보드 리스트의 각 항목은 문서를 식별할 정보(제목·소속 폴더·수정 시각 등)를 표시해야 한다.
- **FR-003**: 사용자가 리스트의 문서를 선택하면 시스템은 에디터 화면으로 이동해 해당 문서를 로드해야 한다.
- **FR-004**: 각 문서는 고유 주소를 가지며, 그 주소로 직접 진입·새로고침해도 같은 문서가 열려야 한다.
- **FR-005**: 존재하지 않는 문서 주소로 진입하면 시스템은 안내하고 대시보드로 돌아갈 수단을 제공해야 한다.

**저장 시 폴더 선택**

- **FR-006**: 사용자는 저장 시 상단 영역에서 저장 대상 폴더를 선택할 수 있어야 한다.
- **FR-007**: 폴더가 선택되지 않은 상태에서 저장을 시도하면 시스템은 저장을 막고 안내를 표시해야 한다.
- **FR-008**: 폴더 선택 후 저장하면 시스템은 해당 폴더 하위에 문서를 저장하고 성공·실패 피드백을 제공해야 한다.
- **FR-009**: 동일 폴더 내 중복 제목은 경고하거나 자동 접미사로 구분해야 한다(권장).

**공통·회귀**

- **FR-010**: 미저장 변경이 있는 상태에서 대시보드·다른 문서로 이동하려 하면 시스템은 경고해야 한다.
- **FR-011**: 라우팅·대시보드 도입이 기존 에디터·사이드바·툴바·저장·다중문서 동작을 깨지 않아야 한다(회귀 0).
- **FR-012**: 대시보드 리스트·폴더 선택·네비게이션은 한글 접근성 레이블을 가지며 키보드로 조작 가능해야 한다.

### Key Entities

- **문서(Document)**: M3에서 정의됨(소속 폴더·제목·본문·수정 시각). M6는 새 엔티티 없이 이 집합을 리스트·
  주소로 노출한다.
- **폴더(Folder)**: M3에서 정의됨. 저장 시 대상 선택 대상.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 대시보드 진입 시 저장된 모든 문서가 리스트에 100% 표시된다.
- **SC-002**: 리스트 문서 선택 후 0.5초 이내에 에디터에 해당 문서가 로드된다.
- **SC-003**: 문서 주소로 직접 진입·새로고침 시 항상 같은 문서가 열린다(불일치 0건).
- **SC-004**: 폴더 미선택 저장 시도는 100% 차단되고 안내가 표시된다.
- **SC-005**: 폴더 선택 저장 시 문서가 항상 지정 폴더에 저장된다(오배치 0건).
- **SC-006**: 존재하지 않는 문서 주소 진입이 앱을 깨뜨리지 않고 안전하게 처리된다.
- **SC-007**: 기존 기능(에디터·사이드바·툴바·저장·다중문서·리사이즈·동기화) 회귀가 0건이다.
- **SC-008**: 대시보드·폴더 선택·네비게이션이 키보드로 접근 가능하고 한글 레이블을 노출한다.

## Assumptions

- 라우팅은 requirement.md §2 구조를 따른다: `/`(대시보드), `/editor/[docId]`(에디터). 진입점이 에디터에서
  대시보드로 전환된다.
- 저장 대상 폴더 선택은 상단 영역의 선택 컨트롤로 제공한다. 사이드바의 폴더별 문서 생성(M3)과 공존한다.
- 데이터(폴더/문서)는 M3 저장소를 그대로 사용한다. 새 저장 모델·이관은 없다.
- 실제 디스크 `.md` 내보내기(FSA API)는 본 마일스톤 범위 밖(선택, FR-7.5).
- 시각 토큰·anti-slop 규칙은 기존 확정분을 따른다. 데스크톱 우선, 모바일은 범위 밖(선택).
