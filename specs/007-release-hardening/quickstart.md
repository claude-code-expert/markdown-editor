# Quickstart: M7 마감 검증

전 마일스톤에 흩어진 수동 검증을 흡수한 **정식 수용 시나리오**. 자동(Vitest)·수동(브라우저) 구분.

## 사전

```bash
npm test          # 기존 22파일 + M7 신규(대비·성능·가드) 전부 GREEN
npm run build     # 무설정 빌드 성공(FR-012, SC-008)
npm run dev       # 수동 시나리오용
```

## 검증 매트릭스 — 무엇이 자동/수동인가

| 영역 | 자동(Vitest) | 수동(브라우저) |
|------|--------------|----------------|
| 디바운스 1회(P1) | ✅ render-budget/타이머 | — |
| 렌더 예산 1만자(P3) | ✅ render-budget | — |
| 입력 체감 즉시성(P2) | — | ✅ M1 |
| IME 안정(P4) | — | ✅ M2 |
| 색 대비(A4) | ✅ contrast.test | 시각 확인 |
| 키보드 도달·리사이즈(A1·A3) | 부분(라벨 존재) | ✅ M3·M4 |
| dirty 가드 3경로(S1) | ✅ 컴포넌트(대시보드) | ✅ M5 |
| sanitize XSS(S2) | ✅ sanitize.test | — |
| 토스트 구분(S3) | 스냅 | ✅ M6 |
| 영속(S4) | storage 단위 | ✅ M7 |

## 수동 시나리오

### M1 — 성능 체감(P2)
1. 1만 자 텍스트 붙여넣기 → 중간에서 연속 타이핑. **기대**: 입력 끊김 없음, 프리뷰는 멈춘 뒤 갱신.

### M2 — 한글 IME(P4)
2. 한글을 빠르게 입력. **기대**: 조합 깨짐·중복 없음, 프리뷰 떨림 없음.

### M3 — 키보드 전용(A1·A3)
3. 마우스 없이 Tab 순회 → 모든 컨트롤 포커스 표시. **기대**: 툴바·트리·핸들·폴더선택·대시보드 도달.
4. 리사이즈 핸들 포커스 → `←`/`→`(±5%p), `Home`/`End`(최소/최대). **기대**: 경계 이동,
   하한(180px·25%)·상한(480px)에서 정지.
5. 스크린리더(VoiceOver)로 각 컨트롤 한글 이름 청취.

### M4 — 영역 리사이즈 시각(A3)
6. 사이드바·에디터:프리뷰 경계 드래그. **기대**: 한계에서 멈춤, 렌더 정상.

### M5 — 안전 종료(S1)
7. 편집(dirty) → 다른 문서 클릭 → 경고. 취소 → 내용 유지.
8. dirty 상태 → **대시보드 링크 클릭** → 경고(가드 신규). 취소 → 에디터 유지.
9. dirty 상태 → 새로고침 → 브라우저 이탈 경고.

### M6 — 토스트 구분(S3)
10. 저장 성공 → 성공 토스트(좌측 `--ok` 보더). 실패 유도 → 실패 토스트(좌측 `--danger` 보더).
    **기대**: 둘이 시각적으로 구분. 컬러 그림자·글로우 없음(헌법 V).

### M7 — 영속(S4)
11. 저장 → 새로고침 → 동일 폴더/문서/내용 보존. 12. 탭 닫고 재방문 → 보존.

### M8 — XSS 시각 확인(S2)
13. 본문에 `<script>alert(1)</script>`, `<img src=x onerror=alert(1)>` 입력. **기대**: 프리뷰에서
    실행 안 됨(alert 0회), 텍스트만 안전 렌더.

## 통과 기준

- 자동: `npm test` 전부 GREEN(회귀 0, SC-009), `npm run build` 성공(SC-008).
- 수동: M1~M8 13개 항목 전부 기대 충족. 미충족 = spec/구현 보강 후 재검증.
