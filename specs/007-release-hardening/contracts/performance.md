# Contract: 성능 (US1 / FR-001~003)

성능 마감 계약. 기준값 = requirement.md NFR-2.

## P1 — 프리뷰 디바운스(FR-001, SC-002)

- 입력 중 **매 글자마다 재렌더하지 않는다**. 입력이 멎은 뒤 **≤150ms** 디바운스 후 1회 갱신.
- 현 구현: `Preview.tsx:17` 120ms `setTimeout` 트레일링 → 충족.
- **계약 테스트**: 연속 변경 N회를 타이머 mock으로 발생시키면 렌더 콜백은 **정확히 1회**.

## P2 — 입력 표시 즉시성(FR-002, SC-001)

- 1만 자 문서에서 입력한 글자는 **1프레임(~16ms)** 안에 에디터에 표시(끊김 없음).
- 입력 경로(CodeMirror 6)는 프리뷰 재렌더와 분리 → 큰 문서에서도 타이핑이 렌더 대기에 묶이지 않음.
- **측정 한계**: 프레임 단위 체감은 jsdom 단위 테스트로 정밀 측정 불가 → **quickstart 수동**으로 검증.

## P3 — 렌더 예산(보조 자동 검증)

- 1만 자 마크다운에 대한 `renderMarkdown` 1회 실행이 환경 보정 예산(jsdom 기준 ≤50ms 가이드) 내.
- **계약 테스트**: `render-budget.test.ts` — 1만자 입력 생성 → 렌더 시간 측정 → 예산 단언.
- 목적: 파이프라인이 문서 크기에 비선형 폭증하지 않음을 회귀로 고정(절대 ms는 환경 의존이라 가이드값).

## P4 — 한글 IME(FR-003)

- IME 조합 중간 상태가 프리뷰를 깜빡이게 하거나 입력을 가로채지 않는다.
- CodeMirror 6가 IME를 내부 처리(별도 compositionend 핸들러 불필요, TRD "한글 IME 안정").
- **검증**: quickstart 수동(한글 빠른 타이핑 시 조합 깨짐·중복·프리뷰 떨림 관찰).

## 측정 명시(no silent cap)

자동 검증 = P1(디바운스 횟수)·P3(렌더 예산). 수동 검증 = P2(체감 즉시성)·P4(IME). quickstart에
"무엇이 자동/수동인지" 표로 명시한다.
