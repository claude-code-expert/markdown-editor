## User Scenarios & Testing *(mandatory)*

### User Story 1 - 모든 마크다운 태그를 툴바로 삽입 (Priority: P1)

작성자가 M1에서 빠졌던 마크다운 문법(굵은 기울임·참조 링크·자동 링크·강제 줄바꿈·Setext 제목)을
문법 암기 없이 툴바 버튼으로 삽입한다. 프리뷰는 이미 모든 문법을 정확히 렌더하므로, 본 스토리는
"삽입 수단"을 완성해 마크다운 기능 명세(코어 + GFM)의 전 항목을 툴바로 다룰 수 있게 한다.

**Why this priority**: M2의 핵심 목표 "플러그인 완성". 명세가 규정한 전체 태그를 툴바로 덮어야
"문법 암기 불필요"라는 제품 약속이 완성된다.

**Independent Test**: 각 신규 버튼을 눌러 올바른 마크다운 문법이 삽입되고, 프리뷰가 해당 결과를
정확히 렌더하는지(굵은 기울임 → 강조 중첩, 참조 링크 → 링크, 자동 링크 → 링크, 강제 줄바꿈 →
줄바꿈)로 단독 검증된다.

**Acceptance Scenarios**:

1. **Given** "강조"를 선택, **When** 굵은 기울임 버튼 클릭, **Then** `***강조***`로 감싸지고 다시
   누르면 해제된다.
2. **Given** 텍스트 선택, **When** 참조 링크 버튼 클릭 후 식별자·URL 입력, **Then** 본문에
   `[텍스트][id]`가 삽입되고 문서 하단에 `[id]: URL` 정의가 1건 자동 추가된다.
3. **Given** URL 텍스트 선택, **When** 자동 링크 버튼 클릭, **Then** `<URL>`로 감싸진다.
4. **Given** 줄 끝에 커서, **When** 강제 줄바꿈 버튼 클릭, **Then** 줄 끝에 줄바꿈 마커가 삽입되어
   프리뷰에서 줄이 바뀐다.
5. **Given** 한 줄 텍스트에 커서, **When** Setext 제목 버튼 클릭, **Then** 아래 줄에 밑줄 마커가
   추가되어 제목으로 렌더된다.

### User Story 2 - 적용된 서식을 툴바에서 시각 확인 (Priority: P1)

작성자가 커서를 굵게·기울임·코드·취소선 등 서식이 적용된 구간 안에 두면, 해당 툴바 버튼이 활성
상태로 표시된다. 선택·커서를 옮기면 활성 표시가 실시간으로 갱신된다.

**Why this priority**: 토글 UX(이미 적용된 서식 해제)의 마지막 조각. 현재 상태를 눈으로 확인할 수
없으면 토글이 추측 게임이 된다. M1에서 `isActive` 로직은 있으나 툴바에 반영되지 않았다.

**Independent Test**: 굵게 적용된 텍스트 안에 커서를 두면 굵게 버튼이 활성으로 보이고, 평문으로
커서를 옮기면 비활성으로 바뀌는지로 단독 검증된다.

**Acceptance Scenarios**:

1. **Given** `**굵게**` 안에 커서, **When** 툴바 확인, **Then** 굵게 버튼이 활성 상태로 표시된다.
2. **Given** 활성 표시 상태, **When** 평문 구간으로 커서 이동, **Then** 활성 표시가 해제된다.
3. **Given** 제목 줄에 커서, **When** 툴바 확인, **Then** 해당 제목 레벨 버튼이 활성으로 표시된다.

### User Story 3 - 코드 블록에 언어 지정 (Priority: P2)

작성자가 코드 블록을 삽입할 때 언어를 지정하면, 프리뷰의 코드가 해당 언어로 표시된다(언어 클래스).

**Why this priority**: 개발자 대상 제품에서 코드 언어 지정은 가치가 크나, 핵심 삽입 흐름(US1)을
차단하지 않으므로 P2.

**Independent Test**: 코드 블록 버튼으로 언어 "js"를 지정해 삽입하면 본문에 ` ```js `가 들어가고
프리뷰 코드에 언어 표시(클래스)가 반영되는지로 검증된다.

**Acceptance Scenarios**:

1. **Given** 커서 위치, **When** 코드 블록 버튼 → 언어 "python" 지정, **Then** ` ```python ` 펜스가
   삽입되고 프리뷰 코드에 해당 언어 클래스가 반영된다.
2. **Given** 언어 미지정, **When** 코드 블록 삽입, **Then** 언어 없는 펜스가 삽입된다(기존 동작 유지).

---

### User Story 4 - 표 행/열 추가 및 정렬 (Priority: P2)

작성자가 삽입된 표에 행·열을 추가하고 열 정렬(좌/중/우)을 지정한다.

**Why this priority**: 표 편집 편의는 문서 작성 효율을 높이나, 기본 표 삽입(M1)으로도 작성은
가능하므로 P2.

**Independent Test**: 표 안에 커서를 두고 행 추가·열 추가·가운데 정렬을 실행해 올바른 GFM 표
문법(`:---:` 등)이 생성되는지로 검증된다.

**Acceptance Scenarios**:

1. **Given** 표 안에 커서, **When** 행 추가, **Then** 빈 셀로 구성된 새 행이 표 아래에 삽입된다.
2. **Given** 표 안에 커서, **When** 열 추가, **Then** 모든 행에 빈 셀 1개씩 추가되고 구분선도 확장된다.
3. **Given** 표 열 선택, **When** 가운데 정렬 지정, **Then** 구분선이 `:---:`로 바뀌어 프리뷰에 중앙
   정렬로 렌더된다.

---

### Edge Cases

- 참조 링크 식별자가 이미 문서에 존재하면? → 중복 정의를 추가하지 않고 기존 정의를 재사용한다.
- 표 밖에서 행/열 추가를 누르면? → 동작하지 않거나 비활성 상태로 안내한다(오삽입 방지).
- 굵은 기울임을 이미 `***`인 선택에 재적용하면? → 해제(토글)된다.
- 자동 링크 버튼을 빈 선택에서 누르면? → 빈 `<>` 삽입 후 커서를 가운데 둔다.
- isActive 활성 표시 계산이 매우 큰 문서에서 커서 이동을 지연시키지 않아야 한다.
- 강제 줄바꿈을 문단 마지막 줄에 적용하면? → 프리뷰에서 의도대로 줄바꿈되거나(다음 줄 존재 시),
  단독 줄이면 시각적 변화가 없을 수 있음을 허용한다.

## Requirements *(mandatory)*

### Functional Requirements

**누락 태그 — 툴바 완성**

- **FR-001**: 시스템은 굵은 기울임(`***텍스트***`) 버튼을 제공하고, 선택 감싸기·토글 해제를 지원해야 한다.
- **FR-002**: 시스템은 참조 링크 버튼을 제공하여, 본문에 `[텍스트][식별자]`를 삽입하고 문서 하단에
  `[식별자]: URL` 정의를 자동 추가해야 한다. 식별자·URL은 입력 다이얼로그로 받는다.
- **FR-003**: 시스템은 자동 링크 버튼을 제공하여 선택한 URL을 `<URL>`로 감싸야 한다.
- **FR-004**: 시스템은 강제 줄바꿈 버튼을 제공하여 현재 줄 끝에 줄바꿈 마커를 삽입해야 한다.
- **FR-005**: 시스템은 Setext 제목 버튼(보조)을 제공하여 현재 줄 아래에 밑줄 마커(`===` 또는 `---`)를
  추가해야 한다.

**서식 활성 표시**

- **FR-006**: 인라인 서식(굵게·기울임·굵은 기울임·코드·취소선) 및 제목 버튼은 커서/선택이 해당
  서식 구간에 있을 때 활성 상태로 시각 표시해야 한다.
- **FR-007**: 커서 이동·선택 변경 시 시스템은 활성 표시를 실시간 갱신해야 한다.

**코드 블록 언어**

- **FR-008**: 코드 블록 삽입 시 시스템은 언어 지정을 받을 수 있어야 하며, 지정된 언어는 프리뷰
  코드의 언어 표시(클래스)로 반영되어야 한다. 언어 미지정도 허용한다.

**표 편집**

- **FR-009**: 표 안에 커서가 있을 때 시스템은 행 추가·열 추가를 제공해야 한다.
- **FR-010**: 시스템은 표 열 정렬(좌/가운데/우)을 지정할 수 있어야 하며, 올바른 GFM 정렬 문법
  (`:---`, `:---:`, `---:`)을 생성해야 한다.

**공통**

- **FR-011**: 토글 가능한 모든 서식 버튼은 키보드 단축키를 제공하고, 단축키 동작은 버튼 클릭과
  동일해야 한다.
- **FR-012**: 신규 마크다운 태그는 독립 모듈 1개 추가 + 레지스트리 1줄 등록으로 완결되어야 하며,
  마크다운 파서·에디터 어댑터는 수정하지 않는다. (활성 표시·표/언어 입력 UI 같은 툴바 코어 기능
  확장은 태그 격리와 별개로 허용한다.)
- **FR-013**: 모든 신규 버튼·다이얼로그는 한글 접근성 레이블을 가지며 키보드로 접근·실행 가능해야 한다.

### Key Entities

- **참조 링크 정의(Reference Definition)**: 식별자(label) → URL 매핑. 본문 어디서든 참조되며 문서
  하단에 정의 텍스트로 수집된다. 식별자 중복 시 단일 정의 재사용.
- **마크다운 플러그인(Markdown Plugin)**: M1의 기존 계약을 따르는 독립 모듈. M2는 이 집합을 확장한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 마크다운 기능 명세의 코어 + GFM 전 항목이 툴바 버튼 또는 자동 동작으로 삽입·표현
  가능하다(삽입 수단 100% 커버).
- **SC-002**: 커서를 서식 구간 안에 두면 0.1초 이내에 해당 버튼이 활성으로 표시된다.
- **SC-003**: 참조 링크 삽입 시 문서 하단에 정의가 1건 추가되고(중복 시 재사용), 프리뷰에서 링크로
  렌더된다.
- **SC-004**: 코드 블록 언어 지정 시 프리뷰 코드가 해당 언어 표시로 렌더된다.
- **SC-005**: 표 행/열 추가·정렬 지정이 항상 유효한 GFM 표 문법을 생성한다(파싱 깨짐 0건).
- **SC-006**: 신규 태그 추가가 마크다운 파서·에디터 어댑터 수정 없이 모듈 1개 + 레지스트리 1줄로
  완료된다.
- **SC-007**: 모든 신규 버튼이 키보드만으로 접근·실행 가능하며 한글 레이블을 노출한다.

## Assumptions

- M2는 M1의 단일 화면·단일 문서·플러그인 아키텍처를 그대로 계승한다. 폴더·다중 문서·IndexedDB는
  M3 범위로 제외한다.
- 프리뷰 렌더는 M1에서 이미 코어+GFM 전체를 처리한다. M2는 "삽입 수단(툴바)"과 "상태 표시"를
  완성하는 데 집중하며, 새 렌더 파이프라인은 도입하지 않는다.
- 코드 블록 언어 입력은 자유 입력 텍스트를 기본으로 하며, 자주 쓰는 언어를 제안 목록으로 보조한다.
- 참조 링크 정의는 문서 맨 끝에 모아 추가하는 것을 기본으로 한다.
- 코드 구문 하이라이트 테마·수식(KaTeX) 렌더는 본 마일스톤의 필수가 아닌 선택 사항이다.
- 시각 디자인은 M1에서 확정한 디자인 토큰·anti-slop 규칙을 그대로 따른다(신규 버튼도 동일 비주얼).
