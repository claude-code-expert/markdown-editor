# Contract: 참조 링크 플러그인 (`src/plugins/referenceLink.ts`)

전체 문서를 다루는 유일한 플러그인. 본문 삽입 + 하단 정의 수집을 한 번의 순수 `apply`로 수행.

## 동작

```typescript
referenceLink.apply(state, { id, url }): EditorChange
```

1. 선택 위치에 `[선택텍스트(또는 "링크 텍스트")][id]` 삽입.
2. 문서 하단에 `[id]: url` 정의 추가 — **단, 같은 `id` 정의가 이미 있으면 추가하지 않음**.
3. 정의 블록은 문서 끝에 모으고, 본문과 빈 줄 1개로 분리.

## 계약 규칙 (테스트 가능)

| # | 규칙 | 검증 |
|---|------|------|
| R1 | 본문에 `[t][id]` 삽입 | 삽입 위치·문법 |
| R2 | 새 `id` → 하단에 `[id]: url` 1줄 추가(SC-003) | 정의 1건 |
| R3 | 기존 `id` 정의 존재 → 본문만 삽입, 정의 중복 없음(엣지) | 정의 수 불변 |
| R4 | 빈 선택 → `[링크 텍스트][id]` fallback | fallback |
| R5 | 정의 블록이 본문과 빈 줄로 분리됨 | 포맷 |
| R6 | `apply`는 순수(동일 입력 → 동일 출력, 부수효과 없음) | 멱등·순수 |

## 검증 방식

- 기존 정의 탐지: `^\[id\]:\s` 정규식으로 문서 스캔.
- 프리뷰 렌더는 M1 파이프라인이 참조 링크를 이미 `<a href>`로 렌더(추가 작업 없음).
- 기대 HTML 출처: `docs/markdown-editor-spec.md §4.6`.
