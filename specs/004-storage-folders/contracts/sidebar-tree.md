# Contract: 사이드바 폴더>문서 트리 (`FolderTree.tsx`, `Sidebar.tsx`)

M2 사이드바 셸을 실제 트리로 채운다. 워크스페이스 스토어를 구독·호출.

## 상호작용 계약 (테스트 가능 — 렌더 + 스토어)

| # | 규칙 | 검증 |
|---|------|------|
| S1 | `[+]` 버튼(활성) → 폴더 생성 다이얼로그/인라인 입력 → 트리에 폴더 추가 | FR-001·SC-001 |
| S2 | 폴더 행 "새 문서" 액션 → 그 폴더에 빈 문서 + 트리 표시 | FR-002 |
| S3 | 폴더 > 문서 계층이 트리로 렌더(1단계) | FR-003 |
| S4 | 폴더 토글 → 하위 문서 접힘/펼침 | FR-004 |
| S5 | 문서 클릭 → 활성 표시 + 에디터 로드(워크스페이스 selectDocument) | FR-005 |
| S6 | 이름변경 액션 → 폴더/문서 제목 변경·반영 | FR-010 |
| S7 | 삭제 액션 → 확인 후 제거(폴더는 cascade 경고) | FR-011 |
| S8 | 트리 노드·액션 버튼 한글 `aria-label` + 키보드 포커스 | FR-013·SC-007 |
| S9 | 빈 상태(폴더 0) → 안내 + [+]로 시작 가능 | 엣지 |

## 비주얼

- M1 디자인 토큰·anti-slop 준수(그라데이션·컬러그림자 없음). 폴더 아이콘 `Folder`/`FolderOpen`,
  문서 `FileText`, 생성 `FolderPlus`/`FilePlus`(lucide). 활성 문서는 액센트 배경(기능적 상태).
- M2 리사이즈 사이드바 안에 트리 배치(스크롤). [+] 버튼은 M2 비활성 → M3 활성.

## 책임 경계

- `Sidebar` = 헤더([+] 폴더) + `FolderTree` 컨테이너. `FolderTree` = 폴더/문서 노드 렌더·액션 위임.
- 데이터·CRUD는 워크스페이스 스토어. 트리는 표시·이벤트만(스토어 액션 호출).
