# PRD — 마크다운 에디터 웹앱

> **문서 성격**: 제품 요구사항(무엇을·왜). 기술 설계는 `TRD.md`, 원본 정의서는 `requirement.md`, 마크다운 기능 스펙은 `markdown-editor-spec.md`.
> **작성 기준일**: 2026-06-26

## 1. 제품 개요

브라우저에서 동작하는 마크다운 에디터. 좌측 에디터에 입력하면 우측 프리뷰가 실시간으로 렌더되고, 툴바 버튼으로 마크다운 문법을 빠르게 삽입한다. 서버 의존 없이 로컬에 저장하며 Vercel 정적 배포로 동작한다.

**대상 사용자**: 개발자·기술 문서 작성자. 단축키 레퍼런스, README, 기술 노트 등 GFM 문서를 자주 쓰는 사람.

**핵심 가치**
- 입력 즉시 정확한 CommonMark+GFM 프리뷰(WYSIWYG 추측 불필요)
- 툴바로 문법 암기 없이 서식 적용
- 로컬 영속 — 새로고침·재방문 후에도 문서 보존, 계정·서버 불필요

## 2. 범위 — 단계적

마일스톤 기준은 **`requirement.md §12`(canonical, 7단계)**다. 구현은 가치우선으로 진행돼 에디터·플러그인(M4·M5)을 레이아웃·저장소(M2·M3)보다 먼저 완성했다.

```
마일스톤 (requirement.md §12 기준)
├─ M1  프로젝트 스캐폴딩                         ✅ 완료
├─ M2  레이아웃 — 사이드바+컨텐츠 2분할·리사이즈   ⬜ 미착수 ← 다음
├─ M3  저장소 — IndexedDB·폴더 CRUD·[+] 버튼      ⬜ 미착수
├─ M4  에디터/프리뷰 — CM6+unified 50:50·실시간    ✅ 완료(선구현)
├─ M5  플러그인 — MarkdownPlugin + 전체 태그(24)   ✅ 완료(선구현)
├─ M6  문서 흐름 — 폴더 선택 저장·리스트·네비       ⬜ 미착수
└─ M7  마감 — sanitize·접근성·토스트·dirty         🔶 대부분 완료
```

**SDD 매핑**: `specs/001-single-screen-editor` → M1+M4+M7 일부 · `specs/002-full-markdown-toolbar` → M5.
전체 마일스톤 표는 `requirement.md §12`. 아래 FR 표(FR-A~D)는 기능 묶음이며 위 마일스톤 번호와 별개다.

## 3. 사용자 흐름

### 3.1 M1 — 단일 화면 (1차 출시)

```
진입 → (이전 저장본 자동 로드) → 에디터 입력
  → 툴바 버튼 클릭(서식 삽입) → 프리뷰 실시간 갱신
  → [저장] → 로컬 영속 + "저장되었습니다" 토스트
  → [취소] → 마지막 저장 상태로 복원
```

### 3.2 전체 — 폴더 도입 이후

```
[+] 폴더 생성 → 문서 작성 → 폴더 경로 선택 → [저장]
  → 대시보드 이동 → 파일 리스트 → 파일 클릭 → /editor/[docId] 로드
```

참여자별 메시지 시퀀스는 `requirement.md §11`.

## 4. 기능 요구사항

우선순위: **필수**(MVP 차단) / 권장 / 선택. 상세 표(FR-ID)는 `requirement.md §4`.

### FR-A. 에디터 / 프리뷰 — M1 핵심

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| A-1 | 에디터(좌) : 프리뷰(우) 50:50 표시 | 필수 |
| A-2 | 입력 시 프리뷰 실시간 갱신(디바운스 ≤150ms) | 필수 |
| A-3 | CommonMark 0.31.2 + GFM(표·취소선·체크박스) 렌더 | 필수 |
| A-4 | 프리뷰 렌더 전 원시 HTML sanitize(XSS 방어) | 필수 |
| A-5 | 하단 상태바에 문자수·줄수 표시 | 권장 |
| A-6 | 에디터/프리뷰 경계 리사이즈 | 권장 |
| A-7 | 스크롤 동기화 | 선택 |

### FR-B. 툴바

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| B-1 | 마크다운 태그 버튼 렌더(제목·B·I·S·코드·링크·목록·표·구분선 등) | 필수 |
| B-2 | 클릭 시 커서/선택 위치에 문법 삽입·토글 | 필수 |
| B-3 | 인라인 서식은 이미 적용 시 해제(토글, `isActive`) | 권장 |
| B-4 | 링크·이미지는 URL 입력 다이얼로그 | 필수 |
| B-5 | 단축키 매핑(굵게 `Mod-b`, 기울임 `Mod-i` 등) | 권장 |

전체 버튼·아이콘·문법 매핑표는 `requirement.md §4 FR-5`.

### FR-C. 저장 / 로컬 영속

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| C-1 | [저장] 클릭 시 에디터 내용 로컬 영속화 | 필수 |
| C-2 | 저장 성공/실패 토스트 피드백 | 권장 |
| C-3 | 재방문 후에도 문서 보존 | 필수 |
| C-4 | 미저장 변경 후 이탈 시 경고(dirty check) | 권장 |
| C-5 | 실제 디스크로 `.md` 내보내기(File System Access API) | 선택 |

> M1은 `localStorage`로 충족, 폴더 도입(M3) 시 IndexedDB로 이관.

### FR-D. 폴더 / 문서 관리 — 후속

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| D-1 | 사이드바 `+` 버튼으로 폴더 생성 | 필수(M3) |
| D-2 | 폴더 > 문서 트리 표시, 접기/펼치기 | 필수/권장 |
| D-3 | 사이드바 너비 드래그 리사이즈 | 필수(M3) |
| D-4 | 문서 클릭 시 에디터 로드 | 필수(M3) |
| D-5 | 폴더/문서 이름변경·삭제 | 권장 |
| D-6 | 저장 시 대상 폴더 선택(미선택 차단·안내) | 필수(M3) |
| D-7 | 대시보드 파일 리스트 + 클릭 네비게이션 | 필수(M4) |

## 5. 비기능 요구사항

| ID | 항목 | 요구사항 |
|----|------|---------|
| NFR-1 | 보안 | 프리뷰 sanitize 필수 — 사용자 HTML로 인한 XSS 차단 |
| NFR-2 | 성능 | 프리뷰 디바운스 ≤150ms, 1만 자 문서에서 입력 지연 체감 없음 |
| NFR-3 | 접근성 | 툴바 버튼 전부 `aria-label`(한글), 키보드 포커스 가능 |
| NFR-4 | 반응형 | 데스크톱 우선. 모바일은 에디터/프리뷰 탭 전환(선택) |
| NFR-5 | 국제화 | UI 한글 기본, 텍스트 상수 분리로 i18n 여지 |
| NFR-6 | 배포 | Vercel 무설정 빌드 통과, 서버 의존 없이 정적+클라이언트 동작 |
| NFR-7 | 데이터 | 새로고침·재방문 후 문서 보존 |

## 6. 제품 원칙 (구현이 지켜야 할 제약)

- **확장은 파일 추가로** — 새 마크다운 태그 지원 = 플러그인 파일 1개 추가. 기존 코어 수정 없이 기능이 늘어야 한다(자세한 계약은 `TRD.md §3`).
- **정확성 우선** — 프리뷰는 CommonMark 공식 출력과 일치해야 한다. 코어/GFM 확장을 구분 표기한다.
- **로컬 우선·서버리스** — 계정·백엔드 없이 동작. 데이터는 사용자 브라우저에 남는다.
- **디자인 절제** — `.claude/rules/anti-ai-slop.md` 준수. 장식 대신 위계·여백·정렬.

## 7. 성공 기준 (M1)
